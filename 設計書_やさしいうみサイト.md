## 0. 戦略（このサイトが目指す方向）※最優先で読む

- ゴール：水族館専門のおでかけサイトとして日本一
- 競合：じゃらん・アソビュー・いこーよ・水族館.com。彼らの土俵（チケット販売・宿泊予約・口コミ件数）では戦わない
- 武器：実訪問×動画×地図×かわちゃんの人格。この純度を守ることが何よりも優先

### 絶対原則
1. トップページのUIは変更禁止。新規ページはすべて検索から入る「裏の入り口」。導線はフッターとspotページの関連リンクのみ
2. 事実情報（料金・休館日・営業時間・設備）はdata.jsonで中央管理。ユーザーから集めるのは「声」（コメント・写真・行った!チェック）だけ
3. 全ページ data.json → build_pages.py で自動生成。手書きページは作らない
4. かわちゃんの一言・5段階評価は「visited=true かつ verified=true」の館のみ表示。visited（本人の訪問有無）は内部フラグで、表には一切出さない
5. 事実ページには必ず「◯年◯月時点の情報・公式サイトをご確認ください」＋公式リンク。不明は【要確認】のまま消さない
6. 外部サイトの文章・画像は転載しない（事実だけ抜き、説明はかわちゃんのトーンで書き直す）

### ページ拡張ルール
- 生き物別・地域別・テーマ別ページは該当3館以上のみ生成（薄いページの量産禁止）
- 新ページはすべてsitemap.xmlに自動反映

### 看板コンテンツ「かわちゃん流・水族館の楽しみ方」
- 独立ページ1本（行く前／大水槽の見方／ショー／子連れの回り方／おみやげ等の章立て。検索の裏の入り口としても機能させる）
- 各spotページに「この館ならではの楽しみ方」1行フィールドをdata.jsonに追加（例：開館直後にラッコへ直行が正解!）
- いずれも本人承認（verified）必須。AI下書きの一次ソースは過去動画で本人が実際に話している内容とする

### AI検索（LLMO）方針
- AIクローラーは全面許可（robots.txt）。llms.txtをdata.jsonから自動生成し常に最新化
- 方針は「抜かれない工夫」ではなく「引用時に必ず名前が付いてくる工夫」：評価・ランキングは常に「かわちゃんの」「さかなのおにいさん評価」など発信者名とセットで表記し、名前と切り離すと意味をなさないデータ形式にする
- 出典「全国水族館ツアーMAP（さかなのおにいさん かわちゃん）」を全ページ・llms.txt・JSON-LDに機械可読で明記
- onlyフィールドは「日本で◯◯に会えるのは△△だけ」の独立短文で表示（AIが引用しやすい形）
- まとめ・ランキングページの冒頭は結論ファーストの一文から始める

### 公開の順序
品質ゲート（本人承認）→ データ充実 → ページ一斉公開 → 拡散（sitemap提出・シェア画像・アクセス計測）。拡散開始の最終判断はかわちゃん本人

## 1. サイトの思想・コンセプト

### ブランドコア
- 「子どもにも海にも やさしい未来を」
- 難しくなく、楽しく、わかりやすく
- かわちゃんが「一緒に旅している」感覚を大切に

### UXの方針
- **ゲーミフィケーション**：「行った水族館にチェック」「目指せ100館制覇」など、訪問者が能動的に参加できる仕掛けを作る
- **シーズン対応**：夏休み・赤ちゃん連れなど「今すぐ役立つ情報」を先頭・目立つ色で表示
- **モバイルファースト**：スマホで気持ちよく動くことを最優先。PC表示はおまけではなく、グリッドで自動調整

## 2. カラーパレット（デザイントークン）

すべてCSS変数で管理。`--変数名` で統一して使う。

```css
:root {
  --sea:     #0096c7;  /* メインカラー（海の青） */
  --sea-deep:#023e8a;  /* 深い青（見出し・強調） */
  --sky:     #caf0f8;  /* 薄い水色（背景・ボタン） */
  --sand:    #fff9ec;  /* 砂浜ベージュ（ページ背景） */
  --coral:   #ff6b6b;  /* サンゴのピンク赤（CTAボタン・動画ボタン） */
  --sun:     #ffd166;  /* 太陽の黄色（数字・強調テキスト） */
}
```

### 追加カラー（シーズン別）
| 用途 | ボーダー | 背景 | バッジ |
|------|---------|------|--------|
| 夏休みおすすめ | `#ffb703` | `#fffbea` | `#ffb703` |
| 赤ちゃん連れ | `#f9a8d4` | `#fdf2f8` | `#ec4899` |

---

## 3. フォント

```css
font-family: "Hiragino Maru Gothic ProN", "Rounded Mplus 1c", sans-serif;
```

- **丸ゴシック系**を使うことで「やわらかく・子どもに親しみやすい」印象になる
- Webフォントは読み込み速度優先のため使用しない（システムフォントのみ）
- 見出しは `clamp()` で画面幅に応じて自動調整：`font-size: clamp(1.7rem, 5.5vw, 2.7rem)`

## 4. ヘッダーのつくり方

```
[ グラデーション背景（上：水色 → 下：濃い青） ]
  ↑ 泡アニメーション（CSS animation）がうかぶ
  ↑ かわちゃんのイラストが中央に浮かぶ
  ↑ タイトル・キャッチコピー
  ↑ カウンター（◯館 / 100館 制覇中！）
  ↑ プログレスバー
```

### グラデーション
```css
background: linear-gradient(180deg, #48cae4 0%, #0096c7 60%, #0077b6 100%);
```

### 泡アニメーション（コピペで使える）
```css
@keyframes rise { to { transform: translateY(-120vh); } }
/* spanにborder-radius:50%、background:rgba(255,255,255,.18)、animation:rise を当てる */
```

### カウンター数字スタイル
```css
font-size: 2.1rem;
font-weight: bold;
color: var(--sun);         /* 黄色で目立たせる */
text-shadow: 0 2px 4px rgba(2,62,138,.5);
```

## 5. カードのつくり方

```
┌──────────────────┐
│  YouTubeサムネ    │ ← 16:9 aspect-ratio
├──────────────────┤
│ ☀️ 夏バッジ       │ ← シーズンバッジ（該当時のみ）
│ [都道府県タグ]    │
│ 館名              │
│ キャッチコピー    │
│ 🦈 🐟 チップ     │
│ 🗓 料金・休日     │
│ [動画] [公式] [くわしく] │
└──────────────────┘
```

### カードCSS骨格
```css
.card {
  background: #fff;
  border-radius: 18px;
  box-shadow: 0 3px 10px rgba(2,62,138,.12);
  border: 3px solid transparent;
  transition: transform .15s;
}
.card:hover { transform: translateY(-5px) rotate(-.5deg); }
```

### シーズン別カードの強調
```css
/* 夏 */
.card.summer-pick { border: 3px solid #ffb703; background: linear-gradient(160deg,#fffbea 0%,#fff 60%); }
/* 赤ちゃん */
.card.baby-pick   { border: 3px solid #f9a8d4; background: linear-gradient(160deg,#fdf2f8 0%,#fff 60%); }
```

### ソート順（重要）
シーズン対応のカードは常に先頭に。
```js
filtered.sort((a,b) => {
  const as = a.summer ? 2 : a.tags.includes('baby') ? 1 : 0;
  const bs = b.summer ? 2 : b.tags.includes('baby') ? 1 : 0;
  return bs - as;
});
```

## 8. 個別ページ（spot/*.html）の構成

`build_pages.py` を実行すると自動生成される。

```
[ ヘッダー（サイト名リンク） ]
[ kicker（紹介済み or これから紹介） ]
[ 館名 + 都道府県バッジ ]
[ YouTube埋め込み or サムネ画像 ]
[ キャッチコピー ]
[ 生き物チップ + タグチップ ]
[ 🐟 かわちゃんからの一言（hitokoto） ]
[ ☀️ 夏休み情報（summer） ]
[ 情報テーブル（料金/休日/おみやげ/アクセス/ベビーカー/授乳室/ロッカー） ]
[ ボタン群（公式・SNS・シェア） ]
[ ← MAPにもどる ]
[ 💬 みんなのコメント ]
  ├ YouTubeコメント（動画あり館のみ）
  └ 投稿フォーム（GAS連携）
```

## 9. コメントシステム（Google Apps Script）

### 仕組み
- スプレッドシート（commentsシート）に保存
- `?aquarium=館名` でGETすると個別館のコメント最新20件
- パラメータなしで全体50件（トップページの電光掲示板用）
- NGワードフィルター付き（暴言・URL・動物愛護系の抗議をブロック）

### スプレッドシートの列構成
| A | B | C | D |
|---|---|---|---|
| 日時 | 名前 | コメント | 水族館 |

### デプロイ時の注意
スクリプトを更新したら必ず「新しいバージョン」でデプロイしなおす。URLは変わらない。

## 10. ホスティング構成

| 要素 | 使用サービス |
|------|------------|
| コード管理 | GitHub（honmachi169/aquarium-tour-map） |
| 公開 | GitHub Pages |

### デプロイフロー
```
data.jsonを編集
  → python3 build_pages.py（spot/*.html + sitemap.xml を再生成）
  → git add -A && git commit -m "説明" && git push
  → 1〜2分でaquarium.yasasea.comに反映
```

## 11. 新しいページを同テイストで作るときのチェックリスト

### デザイン
- [ ] CSS変数（`:root`）のカラーパレットをコピーして使う
- [ ] フォントは丸ゴシック系（Hiragino Maru Gothic ProN）
- [ ] ヘッダーは海グラデーション＋泡アニメ
- [ ] カードは `border-radius:18px` + `box-shadow` + hover時に浮き上がり

### コンテンツ
- [ ] キャッチコピーは80字以内・体言止めよりも「！」「？」で引きを作る
- [ ] シーズン情報（summer / baby等）は先頭に目立つ色で表示
- [ ] 「ここだけ！」バッジで希少性を強調
- [ ] かわちゃんからの一言（hitokoto）で人格を出す

### 機能
- [ ] localStorage で「行った/お気に入り」などのユーザー状態を保存
- [ ] SNSシェアボタン（Web Share API + Xフォールバック）
- [ ] OGP（og:image / og:description）を必ず設定
- [ ] YouTubeコメント表示（YT Data API v3）
- [ ] GASコメント投稿フォーム

### SEO
- [ ] `<title>` に「館名 + キーワード + サイト名」
- [ ] `<meta name="description">` に140字以内の説明
- [ ] `<link rel="canonical">` を設定
- [ ] sitemap.xml を生成してSearch Consoleに提出
